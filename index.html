// STAGE DETAILS

What Each Stage Does

01
📦

Source

Triggered on every push to main or PR merge. Clones the repository, validates branch rules, and sets build context.

  • GitHub Webhook
  • Branch protection checks
  • Commit metadata capture
02
🐳

Build

Compiles application code and assembles a Docker image with multi-stage builds to minimize final image size.

  • Multi-stage Dockerfile
  • Layer caching (BuildKit)
  • Image tagging by SHA
03
🧪

Test

Runs unit, integration, and coverage checks. Pipeline fails fast if coverage drops below threshold or tests break.

  • Jest unit tests
  • Coverage ≥ 80% gate
  • Artifact report upload
04
🔒

Security

Scans dependencies for CVEs and runs static analysis. Blocks deploy on critical vulnerabilities.

  • Trivy container scan
  • OWASP dependency check
  • Semgrep SAST rules
05
🚀

Deploy

Rolls out to Kubernetes with canary strategy. Monitors rollout health and auto-rollbacks on error rate spike.

  • Helm chart deploy
  • Canary 10% → 100%
  • Auto-rollback on errors
